# Аналитическое тестовое задание (Ringo / Dzengo)

Аналитика для сервиса микрофинансирования под залог авто: разбор CRM-лидов и сделок,
классификация ассортимента, динамика продаж запчастей, анализ авторынка РБ и геоаналитика
клиентов. Проект выполнен на Python; по каждому блоку — ноутбук с анализом, а поверх собран
сводный PDF-отчёт.

## Стек

- **Анализ:** Python, pandas, numpy
- **Визуализация:** plotly, matplotlib, folium (интерактивные карты)
- **Прогноз:** statsmodels (Holt-Winters)
- **Геокодинг:** Nominatim (OpenStreetMap)
- **LLM-классификация:** polza.ai (агрегатор ИИ-моделей, OpenAI-совместимый API)
- **Отчёт:** reportlab (PDF), Power BI (дашборд)

## Кратко по блокам

- **Блок 1 — Лиды и сделки.** Чистка дублей, связь лидов со сделками, конверсия по источникам,
  портрет целевой аудитории, таблицы (конверсия, суммы/срок, отказы/сегменты), проблемы CRM.
- **Блок 2 — Ассортимент.** Классификация наименований доп.оборудования по 5 группам правилами
  (`classify_task1.py`) и гибридом «правила + LLM с самообучением» (`classify_hybrid.py`):
  нераспознанные наименования уходят в polza.ai, новое правило дописывается в `learned_rules.json`.
- **Блок 3 — Запчасти.** Разбор показателя «доля клиентов без запчастей»: рост в III–IV кв.
  оказался следствием подмены нулей копеечными начислениями (gaming метрики), а не поведения.
- **Блок 4 — Авторынок.** Топ брендов, доли по регионам (VW силён на западе, Renault на востоке),
  сегментация, поквартальный прогноз продаж VW на 2012–2013 (Holt-Winters).
- **Блок 5 — Геоаналитика.** Геокодинг 605 адресов клиентов, карты folium (точки + плотность),
  концентрация на юго-западе Минска, идеи по локации офиса и таргетингу.

## Запуск

1. Установить зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Открыть ноутбуки блоков в Jupyter / PyCharm (`blok_*/blok_*.ipynb`).
3. Собрать PDF-отчёт:
   ```bash
   python report/gen_maps.py         # карты Минска (PNG с подложкой OSM)
   python report/build_report_human.py   # -> report/Отчёт_аналитика.pdf
   python report/build_report.py         # -> report/Итоговый_отчёт.pdf
   ```

## Настройка polza.ai (для гибридной классификации, Блок 2)

LLM-фолбэк использует сервис polza.ai. Ключ и модель задаются в файле `.env` (в репозиторий не
коммитится). Пример:

```
POLZA_API_KEY=ваш_ключ
POLZA_BASE_URL=https://api.polza.ai/api/v1
POLZA_MODEL=openai/gpt-4o-mini
```

Классификация по правилам работает и без ключа; polza.ai нужна только для новых/нераспознанных
наименований.

## Результаты (отчёт)

- `report/Отчёт_аналитика.pdf` — отчёт живым языком (для защиты).
- `report/Итоговый_отчёт.pdf` — формальная версия.
- Оба содержат ответы, таблицы и графики по всем пяти блокам.

## Ограничения

Работа на тестовых выборках (лиды 624, сделки 50, регистрации авто 5999, визиты 4258,
адреса 605). На малых выборках (особенно 50 сделок) часть выводов — обоснованные гипотезы, а не
строгая статистика; в отчёте это отмечено.
